# 043 — BaseHasefer-style Code-page example pack

**Date:** 2026-06-25

## Why

The user pointed at basehasefer.com (a Hebrew Tanach search/analytics site) and its user manual, asking which
of its capabilities we can add. BaseHasefer is essentially a Tanach **search + analytics engine** with a few
visual tools (parsha-stats viewer, frequency viewer, taam parser, text comparator, object clouds) on top. Our
**Code page is the same engine** expressed as SQL + JS — so most of its *searches* become example queries on
what we already have (`gematria()`, `letters()`, `words()`, `strip()`, `evalJS()` regex, the Hebrew-char fns).
This entry adds the first batch — the "example pack" the user chose to start with.

## What shipped

**New SQL function — `gematria_atbash(text)`** (`plugins/gematria/index.tsx`): atbash gematria, summing each
letter's *mirror* across the alef-bet (א↔ת, ב↔ש, …). Registered alongside `gematria()`. Note the platform
namespaces plugin SQL fns by plugin id (`nsName` in `host.tsx`) — a plugin's namesake (`gematria`) stays bare,
but a second fn becomes `gematria_atbash` (like `torah_code_find`). Verified: `gematria_atbash('שלום')` = 112,
`('תורה')` = 174.

**Four new dropdown samples** (each verified against the live DB; they appear in the Code page's sample
dropdown — `allSamples = [...SAMPLES, ...pluginSamples]`):

- **Atbash gematria (each word, vs. standard)** (gematria plugin) — Genesis 1:1's words with both gematria and
  atbash; comment shows how to filter to *search* by atbash value.
- **Words that appear only once in the Torah (hapax legomena)** — `GROUP BY letters(word) HAVING count(*)=1`;
  6,845 words. (BaseHasefer's Frequency Viewer "occurs once".)
- **The letters שעטנז in order (a "shatnez" check)** — `letters(c.text) LIKE '%ש%ע%ט%נ%ז%'`; 20 pesukim. (Pure
  SQL: `LIKE` with `%` between letters matches a subsequence — verified identical to the evalJS-regex version,
  so no evalJS needed. Their שעטנז wildcard-sequence example.)
- **Anagrams of a word in the Torah** — sort letters via evalJS and match permutations; `אבר` → בָּאֵר, בָּרָא.
  (SQL has no array-sort, so evalJS is the right tool; a `sortletters()` helper would be one-off → skipped.
  Their שינויים / anagram transform.)

`cspell.json` got the new domain terms (atbash, shatnez, hapax, legomena, anagram, pesukim, pasuk). `npx tsc`
clean; cSpell clean on the touched files.

## Mapping for later (not built yet)

Bigger BaseHasefer features are separate plugins/data work, deferred pending the user's pick:
**object/word clouds** (fits the existing `render()` cell-renderer infra), **text comparator** (shared terms /
LCS / diff of two sections), **taam parser** (trop parse-tree — needs a ta'amei-hamikra grammar), and
**parts-of-speech search** (needs morphology tagging we don't have; theirs is flagged experimental too). A
parsha pesucha/stuma stats table would need setuma/petucha break data in the ingest.
