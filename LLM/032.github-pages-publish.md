# 032 — GitHub Pages publishing (torah-llm → torah-app)

**Date:** 2026-06-19

## Why
Ship the app publicly on GitHub Pages without committing gigabytes of SQLite to any branch. Keep
**torah-llm the source of truth**; **torah-app** (the old publisher repo) does the publishing. Live at
**https://kolodny.github.io/torah-app/**.

## What shipped
- **Two-repo deploy.** `torah-app/.github/workflows/deploy.yml` clones torah-llm, regenerates the corpus
  from the pinned sources, builds the published subset, and deploys via the **GitHub Pages Actions
  artifact** (`upload-pages-artifact` + `deploy-pages`) — no `gh-pages` branch, no SQLite in git in either
  repo. Replaces the old force-push-to-`gh-pages` flow.
- **Fit under the 1 GB Pages limit.** Full corpus gzips to ~1.4 GB. `ingest/build-publish-master.ts`
  derives a filtered `publish-master.sqlite` (constants: `SOURCES`/`LANGS`/`EXCLUDE_TOP_CATEGORIES` —
  currently Sefaria-only, he+en, dropping **Halakhah + Second Temple**, links pruned to surviving
  endpoints) → **~937 MiB / 19,422 books**. Measured levers: **index-free slices** (slices are write-once
  transport merged into the indexed client cache, so their indexes are dead weight; −10%, always on),
  dropping English (−~280 MB), dropping links (−~190 MB). `slice-db` gained `MASTER=`/`OUT=` overrides;
  vite `base` is `PAGES_BASE`-driven (`/torah-app/`). New scripts: `publish:master/slice/db`.
- **Fast app-only deploys.** The workflow caches `public/db` keyed on `sources.lock.json + ingest/** +
  shared/**`; an app-only change is a cache hit and skips the ~22 min corpus build (deploys in ~2–3 min),
  while a sources/ingest change misses and rebuilds. A disk-reclaim step frees ~40 GB for the full build.
- **Auto-deploy on push.** torah-llm's `.github/workflows/notify-publish.yml` fires a
  `repository_dispatch(torah-llm-updated)` at torah-app on push to main (path-ignoring docs); it no-ops
  gracefully until a `TORAH_APP_DISPATCH_TOKEN` secret (a PAT that can dispatch to torah-app) is set.
- One-time setup: torah-app → Settings → Pages → Source = **GitHub Actions** (done). First run verified
  green; the live site boots, the catalog is the published subset (Halakhah/Second Temple absent), and
  on-demand slice fetch + gzip-inflate + he/en render all work on Pages.
