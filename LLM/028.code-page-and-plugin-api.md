# 028 — SQL Code page + generic page-extension API

**Date:** 2026-06-11

## Why
The plugin platform ([027](027.plugin-platform-v2.md)) needed a real workbench surface, and the
gematria/SQL helpers were getting baked into core. The user's rule: **every core page exposes a generic,
typed registration API; no core page references a specific plugin.** "Pull all the gematria out — there
should be no reference to it in the code page."

## What shipped
- **`plugins/code-search/`** — a Monaco SQL workbench page over the downloaded books (read-only). Core
  stays plugin-agnostic: it renders whatever plugins contribute via slots, never naming one.
- **Page-owned extension API** — `codePageApi(ctx)` → `{ registerRenderer, registerFns, registerSample }`
  with exported types (`CellRenderer`, `CodeSample`, `CODE_PAGE_ID`). Plugins extend the page through this
  with full typing; the pattern generalizes to other pages (a page publishes its own `xPageApi`).
- **Plugin-registered SQL functions** — `ctx.data.defineFunctions(specs)`; the host **namespaces by plugin
  id** (namesake stays bare, e.g. `gematria`; others prefix, e.g. `find` → `torah_code_find`); the worker
  compiles each body via `new Function(...)` + `db.createFunction`.
- **Generic markers through SQL TEXT** (sentinel-tagged so they survive as strings): `link(book, ref[, label])`
  → a clickable viewer link, and `render('<type>', …)` → a plugin **cell renderer**, both decoded in the
  results table (`shared/code-link.ts`, `shared/code-render.ts`; sentinels via `String.fromCharCode(1|2)`,
  not NUL which SQLite truncates). **Auto-linking**: a results row exposing `toc_id`/`ref` links to the
  viewer with no function call.
- Built-ins exposed: `strip(text)` (HTML→text), Hebrew cantillation/letter names as 0-arity SQL fns
  (`PAZER()`, `ALEPH()`, …), `evalJS(expr, …vals)`. A **progress-handler guard** cancels any user query
  that runs past `EXEC_TIMEOUT_MS` so a pathological query can't wedge the single worker.
- **Gematria moved entirely out of the core page** into its own plugin (see [029](029.gematria-torah-code.md)).
