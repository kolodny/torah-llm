# 039 — QA round 2 (regressions from 038 + deferred items + broad re-sweep)

**Date:** 2026-06-23

## Why

A second multi-agent QA sweep (same shape as [038](038.qa-round.md)): two workflows (audit → adversarial
verify), one aimed at the **code 038 just changed** (did the fixes introduce bugs?), one a **broad
re-sweep** of areas covered less the first time + the 6 deferred low items. **55 confirmed** (15/17 +
40/45). The changed-code sweep earned its keep — it caught real regressions from my own 038 fixes.

## Regressions from 038 (fixed)

- **MG showed the wrong verse on book switch.** The rewrite's focus `idx` was never reset, so opening a
  new book from the catalog (ref-less) clamped to the *last* verse of the new book. Now the idx-sync effect
  resets to 0 when the ref is absent or not in the new book's ladder.
- **The worker watchdog orphaned sibling in-flight calls.** Terminating the wedged worker rejected only the
  timed-out call; every other concurrent comlink call (a parallel `getSegment`/`ensureBook` — exactly what
  MG fires) hung forever (stuck spinners). Now `killWorker` rejects **all** in-flight calls (a `pending`
  set) and clears the download dedupe (`ensuring`), so the app fully recovers.
- **A watchdog restart silently lost all plugin SQL functions** (`gematria()`, `torah_code_find()`, …) —
  they live in worker memory. The client now **remembers `defineFunctions` specs and replays them** onto
  the fresh worker. (Verified: `gematria('אבג')` → 6 after a forced restart.)
- **Gematria highlight was wiped the instant you navigated to view it** (038's unmount-cleanup). Reverted;
  instead the search page now reflects an already-active highlight on mount (so the Clear button is always
  reachable) and never clears on unmount.

## Other confirmed fixes

- **Note pins** only showed on the volatile `selected[0]` edition → vanished in multi-column / on reorder /
  for standalone editions. Added `Segment.primary` (true for the first column rendering a ref); the host
  sets it and the notes pin gates on it → one pin per verse, deterministically.
- **Single-level (colon-less) books** paginated one verse per section. `getBookSections` now detects a
  single-level book and collapses it to one section (`getSectionContent`/`getSectionLinks` honor the
  sentinel) — read as one page instead of dozens of one-verse round-trips.
- **StoragePage** swallowed download/clear failures (looked like a no-op) → now accumulates failures and
  shows them; progress reaches N/N.
- **Wipe local DB** now confirms first and surfaces failures.
- **manifest.schemaVersion** is validated against `BOOT_VERSION` — a newer published schema is detected and
  the stale-app catalog refresh is skipped (warn) instead of ingesting an incompatible catalog.
- **torah-code** letter stream is now deterministic (`group_concat(... ORDER BY c.ref)`, and the matrix
  query orders by `ref` not the merge-unstable `id`); empty ELS ranges show a diagnostic, not a blank grid.
- `reader.onDidChange` delivers the current context to late subscribers; `decorateHtml` clamps mark offsets
  (no unclosed `<mark>`); Flow picks the first selected edition that actually has text; peek "Download to
  preview" has a catch; persist-on-gesture listeners are removed after first fire; MG source aggregation
  excludes self-links; the verse-count uses the book's section-name label; `download()` applies its result
  under the stale-book guard.

Implemented via three parallel subagents (worker.ts; StoragePage+App; torah-code+main+host) on disjoint
files, with the cross-cutting core (client.ts watchdog/replay/single-level, the MG idx fix, the
`Segment.primary` plumbing across types/segment/ViewerPage/notes) in the main session.

## Verified
Responsive e2e 7/1; pagination unchanged; MG opens at 1:1 with whole-book Sources (8) + per-source download
buttons; the watchdog now rejects the runaway AND its sibling call (no hang) AND replays plugin fns
(`gematria()`→6 post-restart).

## Still deferred (low, noted)
Standalone-segmentation heuristic remains weak on a one-section window; cross-section scroll can briefly
drift the URL section; `ensureBook` still holds the lease for the whole download (cross-tab stall only);
`clearBook` can drop a kept book's cross-references to a cleared book; the `ensuring` map caches success for
the session; a plugin page activated only by `onPage:<its-own-id>` can't self-trigger; module-global plugin
caches (notes/songs) aren't cleared on HMR. None are user-blocking.
