# 020 — QA pass (7 agents, 2 rounds) + prioritized roadmap

**Date:** 2026-06-09

After the full ingest, ran a multi-agent QA + product-review pass (parallel subagents finding issues;
I drove the browser and implemented fixes). Stopped at 2 rounds — round 2 was largely confirmatory
(diminishing returns): the data/links/content all validated clean, so no further rounds.

## Fixed + verified this pass
- **Embedded Sefaria cross-references (`refLink`) navigate.** Sefaria's text HTML embeds
  `<a class="refLink" href="Numbers.33.16-36">` (its internal ref format, not an app route). A click
  handler (`parseSefariaRef`) now parses it → `?book=&ref=` and routes via react-router (cmd/ctrl →
  new tab). Fixes the user's `Num. 33.16–36` example (→ Numbers 33:16).
- **Verse links: collapsed, type-grouped list** (`VerseLinks`) replacing the all-or-nothing
  auto-expand commentary panel. Grouped by `connection_type` (commentary / reference / targum /
  midrash / quotation / …), synonyms merged, study-relevant types first, each a navigable link — no
  inline text dump. Genesis 1:1's 396 links are now a tidy collapsed summary.
- **Section labels from `meta.sectionNames`** — Talmud reads **"Daf 2a"** (not "Chapter 2a"),
  Shulchan Arukh "Siman", Mishneh Torah mitzvot "Mitzvah", etc. Fixes the heading on **~1,375 of
  5,382 books (26%)**, incl. all of Talmud Bavli. (`getMeta` added; the table shipped but was unused.)
- **Talmud ref sort** — `refNum` parses daf ("2a"<"2b"<"3a"), so dapim order correctly (was: all → 0).
- **RTL** broadened to Aramaic/Yiddish/Judeo-Arabic (Talmud/Targumim render right-to-left).
- **Collapsible catalog** — categories collapsed by default (auto-open the active book's ancestors);
  home renders 15 roots / 0 book nodes instead of ~6,500. Fixes the CRITICAL render/perf finding.
- **Download progress** shows received MB when `content-length` is absent (no more frozen "Loading…"
  on the 20–41 MB slices).

## QA verdict — data is clean (validated; no action needed)
- **Links: 100% resolvable** (whole-table, 1,974,310/1,974,310), **~99%+ semantically correct**, 0
  systematic bugs (no wrong-direction / off-by-one / range-collapse / wrong-book). 18 connection types.
- **Content: 0 corruption** (0 U+FFFD, 0 mojibake, 0 empties), correct nikud/cantillation, tag balance
  99.99%, **aligned with the live Sefaria API** (spot-checked Genesis 1:1, Berakhot 2a). Unbalanced
  tags (156 rows) are upstream Sefaria typos.
- **No live XSS** — 0 `<script>/onerror/onload/javascript:/<iframe>`; HTML vocabulary is only
  b/i/span/br/sup/small/big/u/em/strong/a/img; the 516 `<img>` are benign data-URI diagrams +
  Sefaria's image host.

## Prioritized roadmap (not yet done)

**Hardening**
- **Sanitize the `dangerouslySetInnerHTML` sink** (defense-in-depth — no live threat today, but the
  corpus is re-ingested). DOMPurify allowlist → tags: b,i,em,strong,span,br,sup,small,big,u,a,img;
  attrs: class,dir,href,src,data-commentator,data-label,data-ref; `<img src>` only `data:image/` or
  `https://textimages.sefaria.org/`. Add a CSP carefully (the SQLite-WASM worker needs worker/blob).
  Best at ingest (avoids per-render cost over 2.16M rows).

**Navigation / scale (highest UX leverage)**
- **Catalog search / autocomplete** (he+en titles) — biggest remaining gap at 5,400 books.
- **Verse-list virtualization** — long books (Shabbat 3,807 rows; big Halakhah) render every verse.
- **Split `links` out of each book slice** — links are ~80% of Genesis's 9.2 MB; a separate lazy
  `toc_<id>.links.sqlite` makes first paint ~5× smaller and enables per-type link pagination.

**Robustness**
- Empty-content + unknown-`?book=` states (clear message vs blank viewer / dual error box).
- Friendlier errors (today: raw `Error: HTTP 404 for /db/…`); retry on boot-DB failure.
- `CSS.escape` the focused-ref selector; clear the flash `setTimeout`; honor `prefers-reduced-motion`;
  global `:focus-visible` outline (a11y).
- Leader-handoff hardening (client.ts re-acquire + single-retry `withApi`); `merge()` re-downloads
  every slice (the OPFS "already present" guard is dead because the file is unlinked post-merge).

**Study features**
- Permalink / copy-citation per verse; bookmarks + recents + breadcrumbs (localStorage/OPFS).
- Relative-ref linkification (`cf. v. 19`) via current book/chapter context.
- Edition-URL consistency across cross-ref nav (carry or reset `ed=` deliberately).
- Local FTS5 search over downloaded books.

**Additive data** (genuinely new vs Sefaria)
- Full **Biur** (Wikisource dump), more **Open Siddur** siddurim, **PocketTorah audio** (sibling
  `audio` table + player; confirm license). Orayta/Dicta-Library "full" mostly duplicate Sefaria — not
  recommended (see [019](019.full-ingest.md)).

## QA method note
Round 1 (4 agents): code (link/verse rendering), db + data integrity, catalog/UX at scale, product
suggestions. Round 2 (3 agents): per-address-type rendering, link semantic quality, content
safety/fidelity. The Playwright browser is a single shared instance, so agents did static code + SQL
data review while the main thread drove the browser and applied fixes.
