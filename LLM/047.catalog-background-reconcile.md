# 047 — Catalog renders from cache; reconcile moved off the boot path

**Date:** 2026-06-26

## Why

The catalog used to block on the network during boot: `bootSequence` ran `await reconcile(pool, fresh)` before
the first `getToc()` returned, so on **slow / flaky service** the app shell loaded but the catalog "never
progressed" until the manifest check (and any catalog re-stream) finished. The cached catalog is a complete,
valid published snapshot, so it should render immediately and the freshness check should happen in the
background.

## What shipped

- **Boot no longer reconciles (cached case).** `bootSequence` (non-fresh) now just opens the cached DB +
  migrates and returns — the first `getToc()` resolves from cache instantly, regardless of the network. (Fresh
  first-boot still downloads + `reconcile(true)` to record its `publishId`; with the SW precache from
  [045](045.offline-catalog-precache.md) even that is served from cache.) `src/db/worker.ts`.
- **Background reconcile, off the critical path.** New `reconcileCatalog()` worker API runs the manifest
  check + (only if a new publish shipped) the in-place catalog re-merge, **once per session**, and returns
  whether the catalog actually changed. It's `try/catch`-wrapped so slow/offline service just keeps the cache.
- **Apply a new publish with a single reload.** `getToc()` (client) kicks off `reconcileCatalog()` after the
  cached catalog is in hand; if it reports a change, the client `location.reload()`s once to show the merged
  catalog. No new publish → nothing happens. (Simpler than the live-`onCatalogUpdated`-subscription approach I
  first tried, per review — the cached catalog is correct meanwhile, so a reload-on-change is enough.)

## Verified

- **Bad service** (manifest + `db.sqlite` delayed 8 s via route): catalog renders in **~800 ms from cache**
  instead of blocking ~8 s; reconcile runs afterward in the background.
- **No reload loop** (sessionStorage load counter = **1** after a fresh visit): fresh boot records the
  `publishId`, so the background `reconcile(false)` finds it unchanged → `changed=false` → no reload. Warm
  reloads are stable too.

`npx tsc` clean.
