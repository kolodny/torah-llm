# 051 — Code page: a SELECTED scope selector, full sample audit, shatnez + Verse↔Rashi redesign

**Date:** 2026-06-28

## Why

While verifying the [strip fix](049.strip-mesorah-markers.md) I audited **every** Code-page SQL sample against
the live DB, and the user flagged two that were conceptually wrong, plus asked for a way to toggle which books
the samples search (like the Storage/Gematria chip tree).

## What shipped

- **`SELECTED` scope selector.** A reusable `BookCheckTree` (the same Tanakh-division chips used by Storage and
  Gematria) sits behind a "Scope: N books ▾" toggle on the Code page, defaulting to (and falling back to) the
  **Chumash**, persisted to the URL (`books=…`). Before a query runs, the bare token `SELECTED` is expanded into a
  subquery of the chosen book ids — usable anywhere a table/list goes (`c.toc_id IN SELECTED`,
  `JOIN SELECTED s …`). No worker changes; works with the read-only query path. Every multi-book sample was
  rewritten from a hardcoded `('Genesis', … 'Deuteronomy')` list to `SELECTED`; single-verse demos (Gen 1:1,
  triangular, atbash) and all-downloaded scans (largest books, most double-cantillation) keep their scope.
  Added to the help text + autocomplete.
- **Shatnez sample redesigned.** Was a contrived "letters of שעטנז spread across five words in order". Now: a
  single **word containing all five crown-letters** (the שעטנ"ז that receive tagin) — which is the word
  שַׁעַטְנֵז itself, returning exactly **Leviticus 19:19** and **Deuteronomy 22:11**, the verses that forbid the
  mixture. (`evalJS('"שעטנז".split("").every(ch => value.indexOf(ch) >= 0)', letters(word.value))`.)
- **Verse↔Rashi sample redesigned.** Was "any verse + any Rashi-comment in the same book sharing a gematria" —
  coincidental cross-pairs that *looked* like a verse and its own Rashi. Constraining it to the Rashi **on that
  verse** at whole-comment level is empty (a comment's gematria ~never equals its verse's — measured 0/4023).
  The meaningful, common form is **a word in the verse ↔ a word in the Rashi on it**, equal gematria, excluding
  the repeated lemma — e.g. Genesis 30:8 `נִפְתַּלְתִּי` ↔ Rashi `נִתְפַּלְתִּי` (both 970): Rashi's own wordplay. The
  Rashi ref `ch:verse:n` is reduced to its verse `ch:verse` to do the join.

## Verified

Every sample run against the live DB (39 Tanakh books + Rashi on Genesis):
- Claims hold — Gen 1:1 = 7 words/28 letters; 2701 = 73rd triangular; Numbers 7 = 89 verses (longest chapter);
  Exodus 32:1 = 34 words; וַיְדַבֵּר…לֵּאמֹר ×69; top word אֵת ×2622; double-cantillation = Decalogue (Ex 20 + Dt 5)
  + Gen 35:22 + Gen 5:29, poetic books (Psalms/Job/Proverbs) lead per-book; most-referenced = Genesis 1:1;
  376 = עֵשָׂו/שָׁלוֹם/…; atbash runs. (hapax 6845→**6843** — the strip fix correctly stopped counting `{פ}`/`{ס}`.)
- New samples through the actual Code-page UI + `SELECTED` expansion: shatnez → 2 rows (Lev 19:19, Deut 22:11);
  Verse↔Rashi → populated (Gen 30:8 נפתלתי↔נתפלתי …). `npx tsc` clean.
