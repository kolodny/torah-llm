# 021 — QA round 3: real-loading bugs the data QA missed

**Date:** 2026-06-09

Round-2 QA validated the *data* (links resolvable, content clean) but never *loaded a book in the
browser* — so it missed bugs that only manifest at load/navigate time. The maintainer immediately hit
one (`?book=Ba'al HaTurim on Genesis` → error page). Round 3 (4 agents: regression review, a11y,
worker/concurrency, product) + hands-on browser testing caught and fixed them.

## Critical bugs fixed (verified in-browser)
- **Apostrophe breaks `ATTACH` → 564 books unloadable.** `encodeURIComponent` does NOT escape `'`,
  so a book id like `Ba'al HaTurim` produced `ATTACH DATABASE '/toc_Ba'al…'` — the literal `'` closed
  the SQL string. Fixed: escape `'`→`''` in `worker.ts merge()` (the slicer already did this).
  **564 content-bearing books** (every one with an apostrophe) were affected. Verified: Ba'al HaTurim
  on Genesis now loads.
- **refLink navigation broken for ~97% of links.** My earlier fix parsed the `href`, but 97% of real
  refLink hrefs are Sefaria-internal slugs (`/Tosafot_on_Bava_Kamma.22b.2.1`, underscores) that my
  guard rejected → the raw `<a href="/…">` did a hard SPA-404. Fixed: parse the **`data-ref`**
  attribute (canonical, e.g. `"Tosafot on Bava Kamma 22b:2"`) instead, and always `preventDefault` on
  refLinks. Verified both forms: `Deuteronomy.1.19` and the leading-slash `/Deuteronomy.6.4-9` →
  navigate + load correctly. (Lesson: I'd tested the one exceptional case, not the rule.)

## Worker reliability (R3-C; cause "book won't load" too)
- **C2** — reserve SAH-pool capacity (`pool.reserveMinimumCapacity(8)`); default holds ~1–2 DBs but
  `merge` needs boot + slice + temp slots at once → nondeterministic first-download failures.
- **C1** — boot is now retryable (re-arms on failure) instead of permanently poisoning the worker on
  a transient/offline fetch error.
- **M1** — `ensureBook` no longer caches *rejections* (a transient failure used to block a book until
  full reload).
- **H3** — explicit-column merges (`INSERT … (cols) SELECT cols`) instead of `SELECT *`, removing a
  silent-corruption path if the schema is ever reordered.

## Other fixes
- **Catalog collapse bug (R3-A):** a manual category-collapse permanently overrode auto-expand;
  now `openPath` (active book's ancestors) always forces-open so the selected book stays visible.
- **a11y quick wins (R3-B):** global `:focus-visible` ring (controls were borderless → invisible
  focus); darkened `--muted` (#8a8378→#6b655c) for WCAG-AA contrast on the metadata/verse-number text.
- **Link label:** merged `ein mishpat / ner mitsvah` (54,959 links) into the "Ein Mishpat" group.

## Lesson for future QA
Data-level validation (SQL on master.sqlite) is necessary but **not sufficient** — it confirms rows
exist, not that the app *loads and renders* them. QA must actually open books + click links in the
browser, especially for ids with special characters (`'`, `(`, `,`, `/`) and for each href form.
