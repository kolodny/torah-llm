# 044 — Scroll-to-ref fix + clickable logo

**Date:** 2026-06-25

## Why

Two reader fixes:

1. **Linking to a ref didn't reliably scroll to it.** Navigating to `?page=viewer&book=…&ref=…` (a cross-ref
   "open", a shared link) was supposed to scroll the verse into view and flash-highlight it. It worked
   sometimes and missed others — verified with Playwright: `ref=Genesis 30:1` landed in view, but
   `ref=Genesis 15:5` ended up at ~1111 px in an 844 px viewport (below the fold), with the flash class
   applied to an off-screen verse. Reproduced on both desktop and mobile.
2. The **תורה · Torah** title in the header wasn't a link.

## Root cause (scroll-to-ref)

The scroll-to effect used `el.scrollIntoView({ behavior: 'smooth', block: 'center' })`. The smooth animation
is asynchronous, and it **races the section-pagination infinite scroll** (LLM/036): the freshly-loaded
section's top sentinel is within the 800 px `rootMargin`, so the IntersectionObserver fires `loadMore('up')`
which **prepends a section mid-animation**. The browser's in-flight smooth scroll was computed against the old
content, so after the prepend it lands on the wrong offset and the target drifts off-screen.

## What shipped (`src/pages/ViewerPage.tsx`, `src/App.tsx`)

- **Instant scroll:** dropped `behavior: 'smooth'` — `el.scrollIntoView({ block: 'center' })`. Instant
  positioning completes before any prepend, and the existing prepend-adjust (`useLayoutEffect` that pins
  `scrollTop` when a section is inserted above) then keeps the verse fixed as sections load around it.
  Verified: `15:5 / 30:1 / 46:8 / 5:3` all land in view on **desktop and mobile** (iPhone 13).
- **Clickable logo:** wrapped the header `Title` in `<a href={import.meta.env.BASE_URL}>` (→ app home; `/` in
  dev, `/torah-app/` on Pages), `color: inherit; text-decoration: none`. (Wrapping rather than
  `Title component="a"` because Mantine's `Title` doesn't type `href` on polymorphic use.)
