# 027 — Plugin platform v2 (VS Code / WordPress-grade redesign)

**Date:** 2026-06-10

## Why
QA + UX + architecture review (5 parallel agents) found the v1 plugin platform ([026](026.plugin-platform.md))
didn't really serve the "core = DB + renderer, everything else a plugin" vision: `addTextDecorator` was
**dead** (the renderer never applied it), `addView` was all-or-nothing and starved of data, the right
column was panel-XOR-peek, UI actions were silent no-op stubs during `activate`, and `App.tsx` (983 lines)
owned all state. The user chose **"full platform, phased"** + **"rewrite the 4 plugins / clean break"**
(no compat shims), then **"do them all"** autonomously. Done in 5 phases; each gated on `tsc --noEmit`,
`npm run build`, and Playwright in-browser checks.

## What shipped

**Phase 1 — Workbench store** (`src/workbench/store.tsx`). One source of truth for navigation + UI intent
(`book/ref/selectedEditionIds/peek/sidebarPanelId/editorId/selection`), exposed as `useWorkbench() →
{state, dispatch, toast}`. URL-backed fields are **derived from the router's params** (so `<Link>`s, the
Back button, and hand-edited URLs all flow into the same state, no manual sync); ephemeral UI lives in a
local reducer. App + `usePeek` + Verses/VerseLinks/PeekPanel read/dispatch through it.

**Phase 2 — New plugin contract + host** (`src/plugins/{bus,types,host,ui}.tsx`, clean break).
- `bus.ts` — the WordPress-style spine: **actions** (emit/on) + **filters** (apply/add), priorities,
  `Disposable` returns, errors logged not thrown.
- `types.ts` — a **manifest** + one namespaced `PluginContext` (`commands / window / reader / data /
  storage / config / actions / filters`) + **Disposable lifecycle**. (Gone: the flat `api` bag, `PluginApi`,
  `PanelProps & { api }` cast.)
- `host.tsx` — live registry via `useSyncExternalStore`; a **store-backed UI bridge installed *before*
  activation** (kills the silent-stub bug); `Disposable` unload + `import.meta.hot` disposal (kills the HMR
  dup-contribution bug); id-dedupe in the registry; an `apiVersion` gate.
- The 4 example plugins (gematria/search/notes/songs) rewritten against `ctx.*`.

**Phase 3 — Regions + multi-pane + editor registry** (workbench shell). Replaced the hardcoded
`catalog | viewer | side` grid + panel-XOR-peek ternary with a VS Code-style shell: a left **activity bar**
(one button per panel + `activitybar` region items), a **title bar**, a **status bar** (`statusbar-left/right`
region items + a core `BookStatus` item + SQLite version), and a **pane row** where **peek and a plugin panel
coexist** as side-by-side columns. `registerEditor` (with `canRender` scoring) + an **editor picker** replace
the all-or-nothing `addView`: the core registers **Grid** (the parallel-edition reader) and **Flow**
(continuous prose) editors; a plugin can register its own reader the same way. Fixed the StandaloneEdition
double-`onClick` (double-tab) and, structurally, the ghost-peek bug.

**Phase 4 — Structured decorations** (`src/workbench/segment.tsx`). Replaced the dead string→string
`addTextDecorator` with `registerDecorationProvider` returning **`mark`** (highlight a plaintext range, with
an optional click handler) and **`lineWidget`** (a React node above the segment) decorations. `decorateHtml`
splices `<mark>` spans into the corpus HTML at plaintext offsets (preserving tags; verified on **vocalized**
Hebrew — nikud + cantillation). `SegmentText` is now the **single corpus-HTML sink** (grid, standalone, peek)
and **sanitizes with DOMPurify**. Re-decoration is driven by a `decorations.changed` action + book/selection
changes. Proof: gematria **highlights every word matching a searched value** (223 × אֱלֹהִים=86); notes shows
a **📝 pin** on verses that have a note (leading column only).

**Phase 5 — Lazy activation + capabilities.** Manifest `activationEvents` (default `['onStartupFinished']`):
a plugin with only deferred events stays **dormant — contributions absent — until `fireActivation` matches**.
`onBook:<id>` / `onBook:*` fire when a book opens; **songs is dormant until you open a book**. Manifest
`permissions` (`data:read` / `data:write` / `storage` / `network`): an ungranted API **throws**, and
`ctx.data.query` is now **SELECT/WITH/PRAGMA-only** (closes the v1 "read-only by comment" foot-gun). Added a
thin `ctx.config` (persistent, plugin-scoped settings).

## Renderer plugin — `plugins/reader-modes/` (proves a *plugin* can own rendering)
A drop-in plugin (auto-discovered; `data:read`; lazy `onBook:*`) that contributes **two whole reading
views** via `ctx.window.registerEditor`, appearing in the Grid/Flow picker:
- **Tikkun (תיקון קוראים)** — leining mode: a scroll column with nikud + te'amim stripped (consonants only,
  like ktav STaM) beside a vocalized "check" column; toggles for the check column + cantillation persist via
  `ctx.config`. (Decodes HTML entities via the DOM so the columns show clean letters.)
- **Mikraot Gedolot (מקראות גדולות)** — one pasuk centered (He + En) with your chosen commentaries fetched on
  demand (`ensureBook` + `getSegment`) and laid out in columns; verse stepper; a **⚙ Sources selector** scans
  *all* the book's links to list every linked work (grouped by type — commentary / targum / midrash / … —
  filterable, with link counts) so you build your own MG. The choice is per-book + persisted via `ctx.config`
  and defaults to the classic Rashi / Ramban / Ibn Ezra / Sforno / Rashbam / Or HaChaim / Kli Yakar companions.
Verified: picker shows Grid/Tikkun/Flow/Mikraot Gedolot; Tikkun scroll = consonants-only vs. vocalized check;
MG fetched Rashi + Ibn Ezra (real text) for Genesis 1:1. This is the concrete validation of Phase 3's editor
registry — a third party can ship a reader without touching core.

## Verified in-browser (Playwright, cold loads)
Store paths (peek→URL, Back-dismiss, panel = local state, toast, edition add); 4 plugins via the new host
(0 errors); workbench shell + regions + Grid/Flow editor switch + peek-and-panel coexisting; gematria
highlight (223 marks, correct on vocalized text) + notes pin (lineWidget); lazy activation (3 startup plugins,
songs wakes on book open) + SELECT-only `query`. `tsc` EXIT=0; prod build green (**~115 KB gzip**, +DOMPurify).

## Known follow-ups (not done)
- **True code-splitting**: modules still load eagerly (only `activate()` is deferred). A static-manifest split
  would defer module load too. `onCommand`/`onView` activation needs declarative `contributes` so the UI can
  show a command/view *before* its plugin activates.
- **Settings UI**: `ctx.config` exists; no auto-rendered settings panel yet.
- **Decorations**: `mark`/`lineWidget` only (no inline `widget` at an offset — can't inject React into the
  HTML string). Marks crossing a tag boundary rely on DOMPurify to normalize nesting (fine for word
  highlights). The **Flow** editor doesn't apply decorations yet.
- **Mobile**: the multi-pane shell is desktop-first; narrow screens want an off-canvas catalog + stacked panes
  (see the UX backlog from this round and [021](021.qa-round-3.md)).
- The broader UX backlog (catalog search, tree virtualization, recents/breadcrumbs, in-book nav, RTL column
  order, menu Esc/outside-click) is queued but untouched here.
