# 016 — Housekeeping: ingest rename, source pinning, tooltips, defaults

**Date:** 2026-06-09

Small but real changes batched while extending the data model.

- **`scripts/` → `ingest/`.** The folder is the ingestion pipeline; renamed for clarity
  (`package.json` script paths + `tsconfig` include updated). Relative imports unchanged (same depth).
- **Source pinning — `sources.lock.json` (committed).** Records each source's pinned version so a
  clone rebuilds the same data without committing the corpora: git commit SHAs for
  `openscriptures/morphhb` and `MosheWagner/Orayta-Books` (adapters fetch from the pinned ref via
  `ingest/sources/pins.ts` → `githubRawBase`), plus the Sefaria-Export index SHA as a version
  marker for Sefaria's GCS bucket. **Not submodules** — they'd vendor whole repos (Orayta ≈ 300 MB)
  for ~a dozen files and can't represent Sefaria's GCS bucket (not a git repo); the lock file is the
  uniform mechanism across all three. `.gitignore` keeps `/data`, `/public/db`, `*.sqlite` out so
  clones stay small.
- **Edition `info` tooltip.** New `editions.info` column (provenance: version title / source /
  license). Sefaria composes it from version metadata (de-pluralized; drops `unknown` license);
  OSHB/Orayta use static strings. Rendered as a chip `title=` on hover. `BOOT_VERSION` → 5.
- **Default columns: English left, Hebrew right** (per request; reversed from the initial he-left).
- **README** rewritten minimalist, pointing at `LLM/` as the living state (assumes an LLM/human
  reads the work log for depth).
