# 029 — Gematria + Torah-code plugins

**Date:** 2026-06-11 (search perf + samples refined 2026-06-19)

## Why
Exercise the generic page-extension API ([028](028.code-page-and-plugin-api.md)) with real features, and
add Torah-code (ELS — equidistant letter sequences) as both a SQL function and a matrix renderer.

## What shipped
**Gematria plugin** (`plugins/gematria/`):
- Viewer contributions: selection tooltip (gematria of any selected Hebrew), a per-verse value action, and
  a decoration that highlights every word matching a searched value.
- A **Gematria search page** with its own nested book selector (reuses `BookCheckTree`, see
  [030](030.storage-tags-gzip.md)) — search the *selected* books by value (words or whole verses), no URL
  `book` param.
- Registers the `gematria(text)` SQL fn + sample queries, including **verse ↔ Rashi equal-gematria**:
  generic across any book whose `Rashi on <book>` is downloaded (joins on the `Rashi on …` relationship,
  not hard-coded to Genesis).

**Torah-code plugin** (`plugins/torah-code/`):
- Registers `torah_code_find(text, word[, maxSkip][, minSkip])` → a JSON handle `{start, skip, len}`, and a
  `torah-code` **cell renderer** that draws the ELS **matrix** (row width = skip so the word aligns in a
  column; centered window capped to a readable width; each row labelled `book ref`; cells are real viewer
  links). **Find-once composition**: `render('torah-code', book, torah_code_find(...))`.
- **Cross-book ELS** — searches Genesis→Exodus→… as one consonantal stream (the famous תורה Gen→Exo seam
  code renders).
- **Search perf** (so a not-found word over a whole book stays sub-second under the query-timeout guard):
  anchor the scan on the word's **rarest letter** (fewest start candidates), **bail if any letter is absent**,
  and clamp skip to `⌊(n-1)/(m-1)⌋`. (Rabin-Karp/hashing don't help here — the cost is the skip×candidate
  loop, which rarest-letter anchoring minimizes; the early-break already makes per-candidate compare ~O(1).)
