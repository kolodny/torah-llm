# 008 — Commentary & links (on-demand)

**Date:** 2026-06-09

Added commentary and wired verses to their linked content, downloading the linked book on
demand, and skipping any link whose endpoints aren't in the stripped master DB.

## What was added

- **Commentary in the subset:** Rashi on all 5 Torah books (`scripts/fetch-subset.ts`), ~5.6 MB.
  Commentary schemas aren't published separately, so the build falls back to the `sectionNames`
  in each `merged.json` (`['Chapter','Verse','Comment']`, depth 3).
- **`links` table** (`shared/schema.ts`): `from_id/from_ref → to_id/to_ref`, `connection_type`,
  unique on the 4-tuple, indexed both directions.
- **Derived commentary links** (`build-master.ts › buildLinks`): the TOC marks commentaries with
  `dependence:'Commentary'` + `base_text_titles`. A comment ref `a:b:c` maps to its base verse
  by dropping the last segment (`Rashi on Genesis 1:1:2` → `Genesis 1:1`). No link CSVs needed.
  - **Skips links not in the stripped master:** a link is emitted only if **both** the commentary
    and base book have content here *and* the base ref exists. (Result: 7,829 links, 0 skipped
    for the Torah+Rashi subset.)
- **Slices carry links** (`slice-db.ts`): each book slice includes links where it's either
  endpoint; the worker `merge` now also `INSERT OR IGNORE`s `links` (dedups via the unique key).
- **UI** (`App.tsx`): each verse with links shows a `▸ N commentary` toggle. Expanding it groups
  links by book, calls `ensureBook(id)` (downloads that book's slice **on demand** if not local),
  fetches the linked refs, and renders them (Hebrew + English). `client.ts` gained `getLinks`,
  `getContentForRefs`, and `ensureBook` (dedupes concurrent/repeat downloads).

## Two bugs fixed along the way

1. **Slice filename encoding.** Filenames were written percent-encoded (`toc_Rashi%20on%20…`),
   but the dev server **decodes** request URLs before the filesystem lookup → 404 for any title
   with a space. Fix: on-disk name is **raw** (`toc_Rashi on Genesis.sqlite`); the client builds
   the URL/OPFS path with `sliceUrlPath` = `encodeURIComponent(sliceFileName)`, which the server
   decodes back to the raw name. (Genesis/Exodus had hidden this — no spaces.)
2. **Wipe race.** `pool.wipeFiles()` is async; `worker.wipe()` didn't await it, so the page
   reloaded mid-wipe and re-opened the stale DB (which lacked the new `links` table). Fix: make
   `wipe()` await `wipeFiles()`.

- Clicking it fetched **`toc_Rashi%20on%20Genesis.sqlite [200]`** on demand (encoded URL resolved

  "IN THE BEGINNING — Rabbi Isaac said…").
