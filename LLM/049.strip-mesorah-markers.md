# 049 — strip() drops mesorah parsha-break markers ({פ}/{ס}) → letters()/words()/gematria fixed corpus-wide

**Date:** 2026-06-28

## Why

Building the [pasuk-name plugin](048.pasuk-name-plugin.md) surfaced a systemic data bug: Sefaria's "Miqra
according to the Masorah" text embeds parsha-break markers as `<span class="mam-spi-pe">{פ}</span>` (petucha)
and `<span class="mam-spi-samekh">{ס}</span>` (setuma). `stripHtml()` removed the `<span>` **tags** but kept
the bracketed text `{פ}`/`{ס}`, so the plain-text projection ended with an editorial **פ/ס** that isn't part
of the verse. Everything built on `strip()` inherited it:

- `letters()` (= strip + `[א-ת]`) read the marker letter as the verse's **first/last letter** — e.g. Psalms
  24:10 (`…סֶלָה׃ {פ}`) looked like it *ends in פ*, not ה, so it was missing from a "ends in ה" search.
- `words()` emitted a spurious `{פ}` "word"; `gematria`/letter-counts over a whole verse were skewed.

**3,446** downloaded Tanakh verses carry such a marker (all `{ס}`=1982 / `{פ}`=1570, **100%** wrapped in a
`mam-spi-*` span). Not a niche case — it silently corrupted any first/last-letter or per-verse analysis.

## What shipped

- **`shared/strip.ts`**: `stripHtml()` now removes the `mam-spi` spans **with their content** before the generic
  tag-strip:
  ```ts
  const MAM_SPI_SPAN = /<span\b[^>]*\bmam-spi[^>]*>[^<]*<\/span>/gi;
  // …replace(MAM_SPI_SPAN, '').replace(/<[^>]+>/g, '')
  ```
  Targeted by the `mam-spi` class so ketiv/qere spans (`mam-kq*`, which carry **real** text) are left for the
  generic strip to unwrap. One change fixes every consumer — `strip`/`words`/`letters`/`evalJS` SQL fns
  (worker.ts imports `stripHtml as stripTags`) and previews.
- **No data/ingest change, no republish.** The stored HTML keeps the spans (the reader renders raw HTML via
  `dangerouslySetInnerHTML`, so parsha-break styling is unaffected); the marker is dropped only in the
  query-time plain-text projection. Existing downloaded slices work as-is.
- The pasuk-name plugin needed **no workaround** — it already calls `letters()`/`strip()` and is now correct.

## Verified

In-browser (Playwright, 5173, all 39 Tanakh books downloaded), after reload re-instantiates the worker:
- Psalms 24:10 — `strip()` → `…סֶלָה׃ `, `letters()` last = **ה**, `words()` tail = `מֶלֶךְ · הַכָּבוֹד · סֶלָה׃`
  (no `{פ}`).
- pasuk-name `מ … ה`: **93 → 111** matches across 23 books, **Psalms 14 → 17**, and 24:10 now renders.
- Node sanity on the regex: petucha span → `""`, setuma `סלה׃ <…samekh>{ס}</span>` → `"סלה׃ "`, ketiv/qere
  `<…mam-kq…>כתיב</span><…>קרי</span>` → kept (`כתיבקרי`).

`npx tsc` clean. (No test runner in the repo — verified via the live DB + a standalone node check.)
