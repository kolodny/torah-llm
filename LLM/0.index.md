# LLM Index — project state (finger on the pulse)

This is the canonical, always-current index of the `LLM/` work log. It tracks every
`NNN.name.md` item and summarizes **where the project stands right now**. Update it whenever an
item is added or state changes. (Sorts first in the folder, before `001`…)

## What this project is

Multi-source Torah platform that runs **entirely in the browser**. A **canonical**,
source-independent catalog (books keyed by title) is split into small per-book SQLite files; the
app boots from a tiny TOC DB and downloads each book's chunk on demand into persistent browser
storage (SQLite-WASM over the OPFS SAH-pool VFS) — no giant DB. Each book carries many **editions**
(source × language × version), compared side by side.

## Current state (2026-06-10)

**Working, verified in-browser (Playwright):**
- **Plugin platform v2 — a VS Code/WordPress-grade platform** (see [027](027.plugin-platform-v2.md)): a
  **workbench store** (URL-backed state); a manifest + namespaced `PluginContext` + **Disposable lifecycle**
  + an **actions/filters hook bus**; a **workbench shell** with regions (activity bar / title bar / status
  bar), **multi-pane** (peek + a plugin panel coexist), and an **editor registry** (Grid / Flow readers,
  user-switchable) replacing all-or-nothing `addView`; **structured decorations** (`mark`/`lineWidget` +
  DOMPurify) reviving the dead decorator hook — gematria highlight, note pins; **lazy activation**
  (`activationEvents`) + **capability scoping** (SELECT-only `query`). The 4 example plugins were rewritten.
- Pipeline `npm run data` = fetch each **pinned** source (see `sources.lock.json`) → build
  `data/master.sqlite` → slice into `public/db/` (`db.sqlite` boot TOC + `toc_<id>.sqlite` per book).
  Ingestion lives in **`ingest/`** (7 adapters: Sefaria + Sefaria-links, Orayta, Open Siddur, Dicta Talmud, Dicta Library, Wikisource Biur). **Jewish-origin
  sources only** — OpenScriptures morphhb/WLC was removed (Christian-affiliated project); same rule
  excludes STEPBible, OpenScriptures HebrewLexicon, tanach.us/UXLC.
- Content — **the ENTIRE Sefaria corpus incl. complex texts**: **23,441 content books, ~4.11M
  segments, ~3.39M resolvable cross-reference links** (Sefaria's full 657 MB links CSV pruned to
  resolvable — Tanakh↔Talmud↔Midrash↔commentary all navigate). All Tanakh / Talmud (full Shas) /
  Mishnah / Midrash / Halakhah / commentaries / liturgy, merged He+En, **including Sefaria "complex
  texts"** (Abarbanel, Or HaChaim, Kli Yakar, Rav Hirsch, Ralbag, Netziv… — split into node-qualified
  sub-books; see [024](024.complex-texts.md)). Plus overlays: **Dicta** vocalized Talmud, Orayta's
  Rashi/Ramban/Ibn Ezra/Sforno, Dicta responsa, Open Siddur, Wikisource Biur. master 4.1 GB →
  **23,441 slices**; boot DB 7.4 MB. See [019](019.full-ingest.md), [024](024.complex-texts.md).
- Browser app (`npm run dev`): catalog tree, on-demand book download, **bidirectional commentary
  links** as real anchors, **URL state** via react-router (incl. selected editions as `ed=` params),
  scroll-to + highlight the focused ref. **Inline link preview ("peek")**: plain-click a cross-ref to
  preview the target in a sticky side panel — shown **within its section, target highlighted** (URL
  state `?peek=&pr=`, shareable, Back-dismissable); cmd/ctrl-click opens the full page in a new tab.
  See [023](023.inline-link-preview.md), [024](024.complex-texts.md).
- **Editions reader:** side-by-side columns; chips are **reorderable** (dnd-kit) and drive column
  order, **persisted to the URL**; hovering a chip shows provenance (`info`). Default: **English
  left, Hebrew right.**
- **Canonical catalog:** books keyed by title, categories by path — adding a book never renumbers
  others; overlapping books dedupe into one book with selectable editions. See
  [013](013.editions.md) (canonical/editions model — supersedes 012's namespaced ids).
- **Source pinning:** `sources.lock.json` records each source's git SHA / version → reproducible
  clone without committing corpora (`/data`, `/public/db`, `*.sqlite` gitignored → small clones).
  Not submodules (would vendor whole repos; can't represent Sefaria's GCS bucket). See [016](016.housekeeping.md).
- **Cross-tab sharing**: leader-tab owns the SAH-pool worker; a SharedWorker brokers ports so all
  tabs share one DB; Web Locks handles leader handoff. No COOP/COEP headers.
- **Forward-compatible storage — never a full wipe** (see [022](022.forward-compat-storage.md)):
  schema changes migrate the cached `/db.sqlite` **in place** (`CONTENT_MIGRATIONS`, `BOOT_VERSION = 12`
  in `shared/schema.ts`) — the corpus is never re-downloaded to change schema. Content changes are
  diffed per book: a `manifest.json` `publishId` + per-book `content_version` (+ local `book_state`)
  mean only **added/modified books re-download** (replace-merge) and **deleted books are pruned**. All
  four verified in-browser. (User data — notes — deferred to its own never-wiped DB later.)

**Next:** a **notes UI** (the durable data layer + `notes` table already exist — [022](022.forward-compat-storage.md));

**HTML sanitization**, and additive data (Biur/Open Siddur/audio). The [021](021.qa-round-3.md) "open
beside / synced columns" idea can build on the new `peek` URL state.
