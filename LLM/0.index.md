## What this project is

Torah content platform that runs **entirely in the browser**. The Sefaria catalog is split into
small per-book SQLite files; the app boots from a tiny TOC DB and downloads each book's chunk on
demand into persistent browser storage (SQLite-WASM over the OPFS SAH-pool VFS) — no giant DB.

## Current state (2026-06-09)

**Working, verified in-browser (Playwright):**
- Pipeline `npm run data` = fetch (from Sefaria's public GCS bucket) → build `db/master.sqlite`
  → slice into `public/db/` (`db.sqlite` boot TOC + `toc_<id>.sqlite` per book).
- Content: **Torah** (Genesis–Deuteronomy) + **Rashi** commentary, Hebrew + English.
- Browser app (`npm run dev`, http://localhost:5173): catalog tree, on-demand book download,
  he/en reader, **bidirectional commentary links** as real anchors (cmd-click → new tab),
  **URL state** via react-router, **scroll-to + highlight** the focused ref.
- **Stable string keys** (book title / category path) — adding a book never renumbers others.
- **Cross-tab sharing**: leader-tab owns the SAH-pool worker; a SharedWorker brokers ports so all
  tabs share one DB; Web Locks handles leader handoff. No COOP/COEP headers.
- **Self-healing boot DB** via `PRAGMA user_version` / `BOOT_VERSION`.
- **Multi-source, editions model**: overlapping books dedupe into one **canonical** book; sources
  *and* Sefaria's versions/languages become selectable **editions** (Genesis = Sefaria he/en,
  JPS 1917, French, WLC, Orayta — 6). 3 sources via per-source adapters (Sefaria JSON, OSHB OSIS
  XML, Orayta TXT). Reader has an **edition picker** (side-by-side columns). Fetched/built data
  lives under `data/`. See [012](012.multi-source.md), [013](013.editions.md).

**Next:** more editions/sources as desired (e.g. Dicta DOCX); a full import + re-verify at scale;
expand the Sefaria corpus beyond Torah; add Sefaria cross-reference links.

**If the app won't load** (stale browser state from this session's worker rewrites): close all
tabs of localhost:5173 and reload, or DevTools → Application → Clear storage, or "Wipe local DB".
