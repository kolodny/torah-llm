# 031 — Generic catalog placement + Biur nesting fix

**Date:** 2026-06-19

## Why
The Wikisource **Biur** books were surfacing as **top-level orphans** in the catalog. Root cause: the
hewikisource adapter emitted editions/content/links but **no `toc` node**, so build-master's silent
orphan safety-net synthesized bare, unparented `book` rows at the root. The fix exposed a broader smell —
each non-Sefaria adapter placed its books with bespoke, ad-hoc logic that would recur as more Sefaria-absent
works are ingested. The user asked for a robust generic solution (or to skip such works), then refined:
new top-level categories must be declared upfront, and Sefaria must be ingested first so its spine exists.

## What shipped
- **`ctx.category(path)`** on `IngestCtx` (implemented in build-master) — the one way a Sefaria-absent work
  is placed. It grafts onto an **existing** branch: the leaf may be new, but its **parent must already
  exist** (so Sefaria, ingested first, owns the branch). It **refuses to create a new top-level root**
  ad-hoc — those are declared upfront in build-master's **`HOUSE_CATEGORIES`** (currently just
  `Dicta Library`). We trust Sefaria's categories implicitly; ours are explicit.
- **build-master hard-fails** on any book that has content but no toc node (was a silent top-level stub),
  and **asserts Sefaria is the first adapter**. The whole "surfaces at the root" bug class is now
  unrepresentable.
- Refactored **Biur** (hewikisource → grafts onto `Tanakh / Modern Commentary on Tanakh / Biur`, dropped
  the bespoke division sets + Hebrew-name map), **dicta-library**, and **opensiddur** onto the primitive.
- **`publishId` made catalog-sensitive** (`slice-db`) — folds a fingerprint of the shipped TOC spine into
  the hash, so re-parenting / renames / new category nodes also bump the publish and reach already-cached
  clients via `reconcile()` (which `INSERT OR REPLACE`s the toc). Previously only book *content* changes
  bumped it, so a pure catalog change never propagated without a wipe. Verified end-to-end: the live app
  reconciled Biur into place with downloads preserved.
