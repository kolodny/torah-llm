# 013 — Editions (dedupe overlapping books) + data/ consolidation

**Date:** 2026-06-09

Per the owner: overlapping books (three "Genesis") collapse into **one canonical book**; sources —
and Sefaria's many versions/languages — become **selectable editions** (Sefaria's "versions" model).

## Model

- **Canonical TOC**: book ids are now source-independent titles (`Genesis`), categories are paths
  (`Tanakh / Torah`) — no more `source:` prefix. The Sefaria TOC is the spine; other sources attach
  to existing nodes by title (build-master adds a fallback node for any orphan edition).
- **editions** table: `(id, toc_id, source, lang, title, order_index)` — one row per version.
- **content** keyed by `edition_id` (+ denormalized `toc_id`). `links` stay between canonical books.
- `BOOT_VERSION` → 4 (self-heals old boot DBs).

## Sources / editions

- **Sefaria** builds the spine + multiple editions per Torah book: Hebrew (merged), English
  (merged), **JPS 1917**, and **Français (Zadoc Kahn)** — fetched from specific GCS version files
  (enumerated via the GCS listing API; unavailable ones are skipped). Rashi books: he + en.
- **OSHB** adds a `WLC` Hebrew edition; **Orayta** adds an `Orayta` Hebrew edition — both attached
  to canonical book ids by title.
- Result: one `Genesis` with **6 editions** (3 Hebrew: Sefaria/WLC/Orayta; 2 English: Sefaria/JPS
  1917; 1 French). Jonah/Ruth (OSHB), Esther/Song of Songs (Orayta) each appear once at their
  canonical TOC position.

## data/ consolidation

All fetched + built data now lives under **`data/`** (`data/sefaria`, `data/oshb`, `data/orayta`,
`data/master.sqlite`) instead of polluting the repo root; `.gitignore` ignores `/data`.

## Reader UI

The book view shows an **edition picker** (toggle chips); selected editions render as side-by-side
columns per verse (Hebrew RTL, translations LTR). Default = first Hebrew + first translation.

## Bug fixed mid-flight

The worker's `merge` copied content/meta/links but **not the new `editions` table**, so a freshly
merged book showed "pick an edition above" with nothing to pick. Added `editions` to the merge.
(Requires a one-time **Wipe local DB** / clear storage to drop the partial pre-fix cache.)

## Verified (Playwright)

One canonical Genesis (`✓ local · 6 ed.`); toggling WLC + JPS 1917 renders 4 columns
(Sefaria Hebrew | Sefaria English | WLC | JPS 1917) showing the same verse across editions;
commentary still expands; 0 errors; `npm run build` passes.
