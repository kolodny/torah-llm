# 006 — Browser app (SQLite-WASM + React)

**Date:** 2026-06-09

## Files

- `src/db/worker.ts` — SQLite-WASM in a Web Worker on the **OPFS SAH-pool VFS** (persistent;
  no COOP/COEP needed). API (comlink-exposed): `open / exec / exists / ls / remove / wipe /
  download / merge`. `download` streams a remote `.sqlite` into the pool via `importDb`;
  `merge` does `ATTACH` + `INSERT OR IGNORE` of a slice's `content`/`meta`, then unlinks it.
  Also silences the benign "synchronous OPFS unavailable" warning (we don't use that VFS).
- `src/db/client.ts` — spins up the worker once, ensures the boot DB (`/db.sqlite`) is in OPFS,
  and exposes typed helpers: `getToc`, `getLocalCounts`, `getContent`, `downloadBook`, `wipe`,
  `sqliteVersion`. Slice URLs come from the shared `sliceFileName(id)`.
- `src/db/types.ts` — row types mirroring the schema.
- `src/App.tsx` — catalog tree (categories → books) on the left; click a book to download
  (with % progress) + read it; verses grouped by chapter, Hebrew (RTL) beside English.
  Books already in OPFS show "✓ local". Text is Sefaria HTML, rendered as HTML.
- `src/main.tsx`, `src/app.css`, `index.html`, `public/favicon.svg`.

## Config

- `vite.config.ts`: `base: '/'`, `optimizeDeps.exclude: ['@sqlite.org/sqlite-wasm']`,
  `worker.format: 'es'`, `build.target: 'esnext'`.
- `tsconfig.json`: bundler resolution, `esModuleInterop`, `allowImportingTsExtensions`,
  includes `src`/`shared`/`scripts`.
- `npm run build` = `tsc --noEmit && vite build` — **passes** (32 modules; app + worker +
  sqlite3.wasm bundled).

## Data flow

`db.sqlite` (TOC only) loads once into OPFS → user clicks a book → its `toc_<id>.sqlite`
slice is fetched and merged into the boot DB → content read back with SQL. Downloaded content
persists in OPFS, so reloads need no network.
