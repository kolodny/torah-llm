# 019 — Full ingest: the entire Sefaria corpus + the full link graph

**Date:** 2026-06-09

Scaled the adapters from subsets to the full corpus (where additive) and ran the full pipeline.

## Sefaria — entire corpus (merged He + En)
Rewrote `sefaria.ts` to walk the whole `table_of_contents.json`, compute each book's GCS path, and
concurrently fetch its merged Hebrew + English. Refs come from each text's `sectionNames` (covers
Chapter:Verse, Talmud Daf:Line, Mishnah, multi-level commentaries); books with no flat sectionNames
(exotic nested schemas) are skipped. Curated Torah extras (JPS 1917, Zadoc Kahn French) kept.
- Fetched **8,046 merged files / 6,592 books / 3.1 GB**.
- Ingested **6,504 editions across 5,371 content-bearing books**, **842,830** derived commentary links.

## Sefaria links — the full cross-reference graph
New `sefaria-links.ts`: parses the 18-shard, 657 MB links CSV → canonical (book,ref)↔(book,ref). The
`Text 1`/`Text 2` columns give the book titles, so ref = citation minus the title; ranges collapse to
their start verse. Emitted **4,665,907** links; build-master prunes any link whose endpoints lack
content (a new `content(toc_id, ref)` index keeps the prune fast) → **1,974,310 resolvable links**.

## Dicta Talmud — full Shas
`dicta-talmud.ts` auto-discovers all tractates from the repo tree (override map for canonical
spellings — Avodah Zarah, Chullin, Keritot, Menachot, Sotah, Temurah, Zevachim; the Meila/Meilah
duplicate dropped). **37 tractates, 78,187 rows.**

## Build / slice
**master.sqlite = 3.1 GB** — toc 7,855 · editions **6,571** · content **2,163,410** · links
**1,974,310**. Sliced into **5,383 per-book DBs (3.3 GB)** + a 2.2 MB boot TOC (6,523 nodes).
`BOOT_VERSION` → 11. (`data/` + `public/db/` stay gitignored; `sources.lock.json` pins the versions.)

## QA fix — embedded Sefaria cross-references (refLinks)
Sefaria's text HTML embeds its own `<a class="refLink" href="Numbers.33.16-36">` anchors (its
internal ref format), which aren't valid app routes — so they didn't navigate (the user's
Deuteronomy `Num. 33.16–36` example). Added a delegated click handler in `Verses` (`parseSefariaRef`:
book = tokens before the first numeric part; ref = numeric parts joined by ':', range → start) that
routes via react-router (cmd/ctrl-click → new tab). Verified: `Num. 33.16–36` → `?book=Numbers&ref=33:16`
→ Numbers 33:16 resolves. (`cf. v. 19` stays plain text — Sefaria doesn't linkify that relative ref.)
