# 034 — Responsive layout (Mantine AppShell drawers + mobile reader)

**Date:** 2026-06-22

## Why

The hand-rolled three-pane viewer layout (catalog | reader | peek/plugins rail) assumed a wide
viewport. On a phone the three flex columns split the width so the **reader was crushed to ~56 px** —
the app was unusable on mobile, which matters now that it's a published PWA
([032](032.github-pages-publish.md), [033](033.offline-and-reliability.md)) people open on their phones.

## What shipped

Adopted Mantine's responsive system instead of fixed flex panes (commit `0eeb7e8`):

- **Catalog → Drawer on mobile.** Desktop keeps the inline `<nav>` pane; below the 768 px breakpoint
  the catalog becomes a Mantine `<Drawer>` opened by a header **Burger**. Opening a book closes it.
- **Peek / plugins rail → Drawer on mobile.** The aside (inline on desktop) becomes a `<Drawer>`
  opened by a **FAB** (`.aside-fab`) or automatically when a cross-ref preview opens.
- **Header tabs** collapse to icons and scroll horizontally on mobile; **Wipe local DB** moves into a
  menu to reclaim space (`src/App.tsx`, `src/app.css`).
- **Workbench store** gained ephemeral nav/aside **drawer state** (`src/workbench/store.tsx`): opening
  a book closes the catalog drawer; opening a preview opens the aside drawer.
- The desktop experience is unchanged (flex three-pane).

This is also where the **single-`ViewerPage`-tree** structure comes from: `ViewerPage` returns one tree
with the reader at a **stable child position**, rendering catalog/aside as inline panes (desktop) vs
`<Drawer>` (mobile). Keeping the reader at a stable index across the breakpoint is what later let
[033](033.offline-and-reliability.md) eliminate the resize-remount jank — crossing 768 px no longer
remounts the ~1,500-verse reader subtree.

## Verification

Added a Playwright **responsive e2e** (commit `37bd177`): `e2e/responsive.spec.ts`,
`playwright.config.ts`, `npm run test:e2e`. It asserts the mobile layout (Burger opens the catalog
drawer, reader takes full width) and the desktop layout (inline three-pane) — guarding against a
regression back to the crushed-column bug.
