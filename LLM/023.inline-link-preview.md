# 023 — Inline link preview (peek panel)

**Date:** 2026-06-10

## What
Cross-reference links (the `VerseLinks` list items **and** the refLinks embedded in verse/commentary
HTML) now **preview inline** on a plain click instead of navigating away — Sefaria's connections-panel
gesture. **Cmd/Ctrl-click** still opens the full page (in a new tab). The open preview is **URL state**,
so it's shareable and the Back button dismisses it.

## How
- **URL params** `?peek=<book>&pr=<ref>` hold the previewed target (added alongside the existing
  `book`/`ref`/`ed`). A `usePeek()` hook sets/clears them via `useSearchParams` (preserving the rest).
- **`PeekPanel`** renders as a third, sticky column (`.layout.has-peek` → `320px | 1fr | 340px`,
  stacks under ~1100px). It fetches the target on demand through the same version-aware path as the
  reader (`ensureBook` → `getSegment(book, ref)` — a new one-segment query, far lighter than pulling a
  whole book to show one verse) and shows every edition's text for that segment (He/En/…); refLinks
  inside the preview chain to further previews.
- **Click routing** (`opensFull(e) = metaKey || ctrlKey`):
  - `VerseLinks` items stay real `<Link>`s with a correct full-page `href`; a plain click
    `preventDefault`s and calls `setPeek`, while cmd-click falls through to the browser's native
    new-tab on the (correct) href.
  - Embedded refLinks (`onRefClick`) `preventDefault` always (their `href` is a Sefaria-internal slug
    like `/Genesis.3.1` — never followed); plain → `setPeek`, cmd → `window.open(searchFor(...))` with
    the *correct* URL.

## Verified (Playwright, real browser)
- Plain-click a `VerseLinks` item → panel opens (Ba'al HaTurim on Genesis 1:1:1 — the apostrophe book),
  `book` param unchanged (no navigation), `peek`/`pr` added to the URL.
- Plain-click an embedded refLink → panel previews `Genesis 3:1`.
- **Trusted** cmd-click an embedded refLink → `window.open('?book=Genesis&ref=3:1','_blank')` (correct
  URL, not the slug); the inline peek is untouched and the current tab doesn't move. (A *synthetic*
  dispatched click is unreliable here — it bypasses React's handler / `preventDefault` and opens the
  raw slug; only trusted events exercise the real path.)
- Fresh load of `?book=Genesis&peek=Rashi on Genesis&pr=1:1:1` hydrates the panel from the URL alone —
  3 editions (Hebrew/English/Orayta), real text — with the reader rendering Genesis independently.
- Close (✕) removes `peek`/`pr`; the panel disappears. Prod build green.
