# 036 — Section pagination (read one chapter at a time, infinite-scroll)

**Date:** 2026-06-22

## Why

Opening a book loaded the **entire** book at once: `getContent(book)` pulled every row
(`SELECT … WHERE toc_id = ?` — all ~1,500 verses of Genesis × every edition), and the reader rendered
all of them into the DOM. That's slow to first paint, heavy in memory, and was behind the earlier
resize-remount jank ([033](033.offline-and-reliability.md)/[034](034.responsive-layout.md)). The ask:
paginate like Sefaria — never load the whole book; load the section you're viewing and stream in the
rest on scroll.

## How Sefaria does it (reference, read-only)

`static/js/TextColumn.jsx`: the unit is a **section** (a chapter for Tanakh, a daf for Talmud). It keeps
an array of loaded section refs, renders one `TextRange` per section, and on scroll near the top/bottom
loads the **prev/next** section (the API returns `prev`/`next` ref pointers per section). Prepending
preserves scroll position by re-deriving `scrollTop` from the height delta. As you scroll it updates the
URL to the visible `sectionRef`. No virtualization — loaded sections stay in the DOM.

## What shipped

Same model, adapted to this app — where a downloaded book is already **local** (SQLite/OPFS), so a
"section" is one DB query, not a network round-trip. The section key is the ref part before the first
`:` (chapter/daf), matching how the reader already grouped verses.

**Data layer (`src/db/client.ts`)** — four light queries, no worker change (all via `api.exec`):
- `hasLocalContent(book)` — cheap `EXISTS` check; replaces "load the whole book to test presence" (also
  fixed the same waste in the peek panel).
- `getBookSections(book)` — the section ladder: `GROUP BY <chapter>` with verse counts, **text-free**
  (~50 small rows for Genesis), so the reader knows the prev/next ladder without loading any text.
- `getSectionContent(book, section)` — one section's rows across all editions (`ref = ? OR ref LIKE 'sec:%'`).
- `getSectionLinks(book, section)` — links touching one section, same map shape as `getLinks`.

**Store (`src/workbench/store.tsx`)** — new `setRef` action: rewrites only the URL `ref`, `replace`-mode
(no history spam, doesn't touch book/peek) — used by scroll tracking.

**Reader (`src/pages/ViewerPage.tsx`)** — replaced whole-book state with a **windowed section model**:
- On book open: load editions + meta + the section ladder (no verses yet).
- A navigation effect loads the section for the focused `ref` (or the first), resetting the window to it;
  if the target is already in the loaded window it just scrolls there (no reload).
- `loadMore('up'|'down')` grows the window one section at a time, driven by an **IntersectionObserver**
  on top/bottom sentinels bracketing the rendered sections (`rootMargin: 800px` prefetch; the `.viewer`
  pane is the scroll root). Prepend captures `scrollHeight/scrollTop` and a `useLayoutEffect` restores
  position so the page doesn't jump.
- A debounced scroll handler writes the section at the viewport top into the URL via `setRef`. An
  `actedRef` guard makes the navigation effect ignore scroll-driven ref changes (no feedback loop), and a
  `scrolledForRef` guard stops the scroll-to-and-flash from re-firing as the window grows.

Crucially `BookView.content` stays `ContentRow[]` (now the **loaded window**, not the whole book), so the
Grid/Flow core editors and the `reader-modes` plugin (Tikkun / Mikraot Gedolot) keep working unchanged —
the page drives infinite scroll for all of them uniformly via the shared sentinels.

## Verification (`scripts/repro-pagination.mjs` + ad-hoc Playwright)

- Open Genesis → **31 verses (chapter 1 only)**, "31 verses · 1 chapters" — not 1,533. Scrolling grows
  the window 31 → 56 → 80 → 106 → 138; the URL `ref` tracks the section in view (`ref=4`); 0 console errors.
- Deep-link `Genesis 15:5` → loads section 15 **directly** (+ prefetches 14), flashes 15:5; scroll-up
  prepends ch 13 with preserved position; 0 errors.
- Existing responsive e2e: 7 passed / 1 skipped.

## Notes / possible follow-ups
- The "N verses · M chapters" line now reflects the **loaded window**, not the whole book (could show the
  ladder total instead).
- No unloading of far-scrolled sections (same as Sefaria) — fine for books; revisit only if a giant book
  in one sitting bloats the DOM.
