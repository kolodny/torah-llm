# 002 — Design decisions

**Date:** 2026-06-09

## Stable string TOC keys (not auto-increment integers)

`toc.id` is a **string**, chosen by the project owner so that adding a new book to the TOC
does **not** renumber existing keys (auto-increment integers would shift every subsequent
row, invalidating already-sliced files and any saved references).

- Books/texts → the **English title** (Sefaria's canonical, globally-unique identifier).
- Category nodes → a **parent-path** string (e.g. `Tanakh / Torah`).

"Genesis" is always "Genesis" regardless of what else is added to the corpus.

## Slice filenames

`public/db/toc_<encodeURIComponent(id)>.sqlite`. Encoding keeps titles with spaces and
punctuation URL- and filesystem-safe. One shared `sliceFileName(id)` helper is used by both
the slicer (write) and the browser fetch path (read) so they never drift.

## Iteration-1 scope (small + fast)

- Subset: **`json/Tanakh/Torah`** — the 5 books of Moses, Hebrew + English (~30 MB raw).
- **Links skipped** in iteration 1 (the link CSVs are ~476 MB); links are the planned
  "expand" step (a `links` table + per-book link slices).

## Stack (ground-up, minimal)

- **Vite + React + TypeScript**, plain CSS. Dropped MUI / react-query / react-router /
  use-query-params from the reference — not needed for the core demonstration.
- **Raw SQL** over **SQLite-WASM** (OPFS SAH pool) in a Web Worker, exposed via **comlink**.
  Dropped **drizzle / drizzle-kit** — the migration dance was a source of the original
  breakage and adds no value for a small, fixed schema. Schema is one shared SQL string.
- Node ingestion uses **better-sqlite3**; scripts run via **tsx**.
- Base path simplified `/torah-app` → `/` for clean local dev.

## Schema (raw SQL, shared by build + slices)

`toc(id TEXT PK, parent_id, kind, title_en, title_he, category_en, category_he,
order_index, has_content, content_rows, file_size)`,
`content(id INTEGER PK, toc_id, ref, lang, text, UNIQUE(toc_id,ref,lang))`,
`meta(toc_id TEXT PK, schema JSON)`. (`links` added when we expand.)
