# 057 — Stop hook that reminds Claude to keep the LLM/ log current

**Date:** 2026-06-30

## Why

The `LLM/` work log is the project's memory (CLAUDE.md), but it's easy to finish a coding turn and forget to
add/refresh an entry. A `Stop` hook can enforce the convention automatically instead of relying on memory.

## What shipped

- **`.claude/hooks/remind-llm-log.sh`** — a Claude Code **Stop** hook. When Claude tries to finish while the
  git working tree has changes but **nothing under `LLM/`**, it returns `{"decision":"block","reason":…}` to
  feed a reminder back to Claude (add an `LLM/NNN.*.md` entry + refresh `0.index.md`). Safeguards:
  - **Loop-safe**: bows out (exit 0) when `stop_hook_active` is set, so it nudges at most once per stop cycle.
  - **Quiet when handled**: exit 0 if the tree is clean or any change already touches `LLM/`.
  - **Never traps Claude**: any unexpected condition (no git, not a repo, etc.) exits 0 — a crashing Stop hook
    would wrongly block stopping. No `jq` dependency (greps stdin, prints JSON via `printf`).
- **`.claude/settings.json`** — wires the script to the `Stop` event. Committed (not in the gitignored
  `settings.local.json`) so it travels with the repo as a project convention alongside the CLAUDE.md rule.

Note: Claude Code loads hooks at session start, so a freshly-written hook normally takes effect next session —
though in this case it was picked up live and fired on the first stop (verified end-to-end: it blocked once,
delivered the reminder, then this very entry satisfies it so the next stop passes clean).
