# 035 — Cantillation SQL sample (code page example, iterated)

**Date:** 2026-06-22

## Why

The Monaco **SQL Code page** ([028](028.code-page-and-plugin-api.md)) ships sample queries that double
as documentation for its `evalJS` capability. This adds a genuinely hard one — *find a Torah word that
bears two distinct cantillation marks (te'amim)* — and iterates it until it returns only the truly
notable cases. It exercises `evalJS` over Hebrew Unicode and is a good stress test of the sample API;
it is **not** core behavior (no cantillation logic was special-cased into the worker — per the user's
"it shouldn't be special-cased here" correction, it lives entirely as an `evalJS` expression in the sample).

## What shipped (`plugins/code-search/CodePage.tsx`)

Iterated over four commits as spot-checking across the Chumash exposed false positives:

1. **`785c228`** — initial: scan the U+0591–U+05AF range, collect accents into a `Set` so the same
   accent doubled for layout (pashta/telisha) doesn't double-count.
2. **`6dd2187`** — require two **distinct disjunctive** te'amim: drop conjunctives (U+05A3–U+05AA), so
   a servant+disjunctive pair (munaḥ+zaqef, qadma+geresh) no longer counts; merge zinor→zarqa
   (one accent, two positional codepoints); also split on maqaf (־).
3. **`290bcba`** — tokenize on **paseq (׀)** and HTML spacing entities (`&nbsp;`/`&thinsp;`): poetic
   layout (Az Yashir, Ha'azinu) was joining two words into one token and matching them. After this it
   cleanly surfaces the real phenomena — the Ten Commandments' dual **ta'am elyon/taḥton** (Ex 20,
   Deut 5) and the **Reuben pisḳa** (Gen 35:22).
4. **`f5c0f03`** — tidy only: reformat the one-line `evalJS` blob into a readable IIFE with named
   constants and hex codepoints; a single `evalJS` in a subquery rather than duplicated in `WHERE`.
   Same results.

## Takeaway

Hebrew cantillation has several ways to *look* like "two marks" that aren't (conjunctive+disjunctive
pairs, layout-doubled glyphs, multi-codepoint single accents, words merged by spacing/paseq). The
working filter is: tokenize on whitespace + maqaf + paseq + spacing entities, keep only **disjunctive**
accents, normalize zinor→zarqa, and require **≥2 distinct** per token.
