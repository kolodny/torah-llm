# 025 — Cross-source segmentation: standalone-panel rendering

**Date:** 2026-06-10

## Problem
Parallel editions in the reader are paired **by ref** (`2a:1`, `1:1`…), which only lines up when the
sources share segmentation. The maintainer hit a misaligned **Dicta** Talmud column: Dicta segments
Berakhot daf 2a into ~4 chunks (by its source file's line breaks) while Sefaria uses ~14 fine segments
— same `2a:N` labels, **different content**. "Is this elsewhere, and can we generalize a fix?"

## Audit (where it actually bites)
- **Dicta Talmud**: only **2 of 37** tractates (`Berakhot`, `Shabbat`) mis-segment; the other 35 match
  Sefaria 100%.
- **Biur (`hewikisource`)**: a *structural* bug — a commentary ingested as a sparse verse-"edition" of
  the base book (`Genesis`: 80 of 1533 verses) → a mostly-blank column.
- **Orayta**: mostly 100%; one mis-segmented copy (`Ramban on Exodus`) + two Ibn Ezra books at 85–92%
  (genuinely aligned).
- **opensiddur, dicta-library**: standalone (no Sefaria overlap) — no issue.

## Generalized fix — render misaligned editions as a standalone panel (no data dropped)
Nothing is removed from the corpus. The **reader** (`src/App.tsx`, `Verses`) splits the *selected*
editions: the canonical (most refs, Sefaria-preferred) and every edition whose **per-section
segment-count** matches it share the verse grid; an edition whose counts mostly disagree (>50% of
shared sections) is rendered **standalone** below the grid, in its own ref order (`StandaloneEdition`).
The count signal is spelling-independent, so it isn't fooled by Dicta's ktiv-haser vs Sefaria's
ktiv-male — and because it's computed at render time over the *selected* editions, no schema/flag and
no build step are needed.

Effect: open Berakhot with Sefaria He/En + Dicta → He/En align in the grid; **Dicta shows in its own
panel** (its 4-chunk daf), correctly readable. The 35 aligned Dicta tractates and Orayta Ibn Ezra
still render as normal aligned columns. Any future source that mis-segments is handled automatically.

(An earlier iteration had build-master *drop* the misaligned editions; the maintainer preferred keeping
them and showing them separately — no data loss — so the drop guard was removed in favour of this
render-time split.)

## Structural fix — Biur as its own book (`ingest/sources/hewikisource.ts`)
Biur now ingests as **`Biur on <Book>`** commentary books (its verse refs) **linked** to the base
verses, instead of a sparse edition of the base text — so it surfaces as a commentary *connection* on
the verse, not a blank parallel column. (Verified: `Biur on Genesis` is linked at Genesis 1:1.)

## Note
A genuinely *different work* mis-modeled as an "edition" (Biur) belongs in its own book — that's a data
fix. The reader's standalone split is the general safety net for *same-work, different-segmentation*
cases (Dicta), so the columns the reader pairs always line up.
