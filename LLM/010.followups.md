# 010 — Follow-ups & open decisions

**Date:** 2026-06-09

## A. Cross-tab data sharing — RESOLVED (see [011](011.cross-tab-sharing.md))

**Resolution:** implemented option 2 — **leader-tab + SharedWorker port broker**. True sharing,
no cross-origin-isolation headers, verified incl. leader handoff. Details below kept as the
rationale/history.

**Goal:** two tabs share one local DB (a book downloaded in tab A is instantly available in tab B).

**Why the requested Vite plugin can't do it:** the intended architecture is a SharedWorker
(one per origin) owning the OPFS pool. But OPFS sync access handles only exist in a *dedicated*
worker, and **Chrome does not expose the `Worker` constructor inside a SharedWorker** — verified:
the SharedWorker throws `ReferenceError: Worker is not defined`, so it can't even spawn a nested
dedicated worker. This is a browser limitation, not a Vite-dev transform issue, so no bundling
plugin can work around it.

**Current state (shipped):** each tab has its own dedicated worker + its own SAH pool (named per
tab via `sessionStorage`). Multiple tabs work simultaneously with 0 errors; the only downside is
tabs don't share downloaded books (each re-downloads on demand — cheap for our small slices).

**Real options for true sharing:**
1. **COOP/COEP + the multi-connection `opfs` VFS** (not the SAH pool). Each tab keeps its own
   dedicated worker, but they open the *same* OPFS DB file via the `opfs` VFS, which supports
   multiple connections with SQLite locking. Needs cross-origin-isolation response headers
   (all our assets are same-origin, so low impact) and a worker rewrite (swap VFS; replace
   `importDb` with writing bytes to OPFS). Works in dev. **Recommended.**
2. **Leader-tab + SharedWorker port broker.** One tab owns the dedicated worker; the SharedWorker
   only *brokers MessagePorts* between tabs (no worker spawning), so followers talk to the
   leader's worker. Keeps the SAH pool. Complex: leader election + handoff when the leader closes.
3. **Keep per-tab (no sharing).** Current behaviour.
