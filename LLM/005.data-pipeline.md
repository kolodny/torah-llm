# 005 — Data pipeline (fetch → build → slice)

**Date:** 2026-06-09

## Files

- `shared/schema.ts` — `SCHEMA_SQL` (toc/content/meta), the single source of truth.
- `shared/slice-path.ts` — `sliceFileName(id)` = `toc_<encodeURIComponent(id)>.sqlite`, `TOC_DB`.
- `scripts/sefaria.ts` — local-subset readers + `createRefPath`/`format`/`flatten`
  (ported from the reference; fixed a latent double-increment when `index_offsets_by_depth`
  and `sectionNames` are both present).
- `scripts/fetch-subset.ts` — downloads the Torah subset from GCS into `./sefaria`.
- `scripts/build-master.ts` — `./sefaria` → `db/master.sqlite` (full TOC + Torah content + meta).
- `scripts/slice-db.ts` — `master` → `public/db/toc_<id>.sqlite` (per book) + `db.sqlite` (boot TOC).

`npm run data` = fetch → build → slice.

## Keying (the whole point)

TOC ids are **stable strings**: a book's English title (`Genesis`) or a category path
(`Tanakh / Torah`). Assigned directly while walking `table_of_contents.json` — never via
`lastInsertRowid`. Adding a book later cannot renumber existing ids.

## Boot-DB pruning

`build-master` writes the **full** catalog TOC (7850 nodes) to master. `slice-db` copies into
`public/db/db.sqlite` only the content-bearing books **plus their ancestors**, so the boot DB
is tiny and grows only as content is added. (Iteration 1 → 7 nodes: `Tanakh`, `Tanakh / Torah`,
+ 5 books.)

## Verified output (Torah subset)

```
TOC nodes: 7850 (master)        content rows: 11,692 (5 books × he+en)
Genesis 1533 / Exodus 1210 / Leviticus 859 / Numbers 1288 / Deuteronomy 956  (verses per lang)
slices: toc_Genesis.sqlite 856KB … toc_Leviticus.sqlite 524KB
boot db.sqlite: 7 nodes, 36KB
```

DB spot-checks passed: boot TOC tree is connected with `file_size`/`content_rows`; Genesis 1:1
present in Hebrew (with nikud + cantillation) and English; `meta.schema` has
`sectionNames:["Chapter","Verse"]`. Text contains Sefaria HTML markup (`<big>`, footnote
`<sup>`), so the viewer renders it as HTML.
