# 038 — Comprehensive QA round (subagent sweep) + fixes

**Date:** 2026-06-23

## Why

Real-device use surfaced "silly" bugs that the happy-path Playwright checks kept missing — notably, on
mobile with only Genesis downloaded: **Mikraot Gedolot "completely broken"** and **opening a commentary
in the sidebar showed empty / no download prompt**. Rather than guess, ran a broad QA sweep with subagents
across the whole app, then fixed the confirmed issues.

## The sweep

Two parallel multi-agent workflows (audit → adversarial verify of every finding):
- **Hot spots** (recent changes): pagination fan-out, peek/links, Mikraot Gedolot, mobile, the lease.
- **Broad**: every plugin (Code page, gematria, torah-code, search, notes, songs), Storage, the workbench
  platform/bus, navigation/URL/editions, race conditions, offline/PWA/data-integrity.

**49 confirmed bugs** (27→16 + 46→33), the verify pass discarding ~40% as non-issues. The dominant cluster
(11 reports) was Mikraot Gedolot.

## Root cause of the user's reports

Section pagination ([036](036.section-pagination.md)) made `BookView.content`/`links` a **per-section
window**. Mikraot Gedolot still assumed whole-book data, so on open it: (1) built its source list/defaults
from one chapter's links, (2) could only page within the loaded chapter, and (3) **auto-downloaded up to 8
commentary books** (`ensureBook` per source) — silent, sequential, blocking the lease → the "hang". And a
single-verse editor like MG left the infinite-scroll sentinel permanently in the prefetch zone → a runaway
cascade loading every section, re-firing those downloads.

## What shipped

**Mikraot Gedolot rewrite** (`plugins/reader-modes/index.tsx`) — now fully **window-independent**: fetches
its own whole-book verse ladder + linked-source aggregation via `ctx.data.query`, and the focused verse's
base text + links directly via `ctx.data.getSegment`/query. **No auto-download** — selected-but-missing
sources show an explicit per-source "↓ Download to show" button (matching the app's no-autodownload policy);
load failures are surfaced with retry instead of silently dropped. Declares `managesOwnScroll`.

**Host (`ViewerPage`)** — skips the infinite-scroll sentinels for `managesOwnScroll` editors (no more
runaway cascade); whole-book verse/chapter **totals** in the count line (not the loaded window); the **Flow**
reader gained the edition bar; **race guards** so a rapid book switch can't apply a stale prior book's
skeleton (`applySkeleton` under a not-cancelled guard) or append stale rows mid-`loadMore` (a `bookRef` check).

**Worker hang recovery** (`client.ts`) — a synchronous runaway inside a SQL function (`evalJS` or a plugin
fn) can't be preempted by the progress-handler timeout, wedging the single worker forever. A main-thread
**watchdog** terminates + recreates the worker if an `exec` exceeds the timeout + grace (~16 s) and rejects
the call; the next call re-boots a fresh worker. (Verified: infinite-loop query → rejects → app recovers.)

**Plugin platform / others:**
- `readOnlyQuery` SELECT-only guard hardened against multi-statement injection (`SELECT 1; DELETE …`).
- `hasLocalContent` exposed on `PluginData` (gates the MG downloads).
- Failed `activate()` now disposes whatever it half-registered (no partial-activation leak).
- Search: gate on local presence (download prompt vs "0 matches"), dedupe by ref, match/snippet on
  stripped text, escape LIKE wildcards.
- Gematria: clear the global highlight on unmount, ignore value 0, one Hebrew edition per verse.
- Storage: preserve checkbox edits made during a download/clear run.
- Worker: clean up a partial OPFS slice on a failed download; don't wipe a good cache when a *freshly*
  downloaded catalog is corrupt; clamp download progress (no bogus >100% on gzipped slices).
- Code page: try/catch around custom cell renderers (one bad cell no longer blanks the table); don't
  auto-link encoded marker values.
- `navigator.storage.persist()` also retried on first user gesture.

Implemented partly via parallel subagents on disjoint files (worker/client; search/gematria;
host/main/CodePage/Storage), with the MG + ViewerPage core done in the main session.

## Verified
Responsive e2e 7/1; pagination unchanged; MG shows whole-book Sources (8) + per-source download buttons +
**0 auto-downloads** + prev/next spans the book; infinite-loop query restarts the worker and the app recovers.

## Deferred (low severity, noted not fixed)
Standalone-segmentation heuristic is weak on a one-section window; single-level (colon-less) books paginate
one verse at a time; cross-section scroll can briefly drift the URL section; `manifest.schemaVersion` not
validated; a few torah-code letter-stream/group_concat edge cases; `ensureBook` holds the lease for the
download (cross-tab stall only).
