# 052 — Cleanup functions audited corpus-wide: strip footnotes/annotations; gematria + Torah-codes routed through strip

**Date:** 2026-06-29

## Why

Spot-checking the cleanup functions against known facts (Torah letter count, famous ELS codes) exposed two real
bugs: `gematria()` and `torah_code_find()` extracted letters from **raw HTML** with a *disallow* regex
(`/[^א-ת]/`), bypassing `strip()`. A disallow regex can't tell a real פ from the `{פ}` parsha marker (both are
Hebrew letters), so:

- **Corpus-spanning Torah codes were wrong.** `torah_code_find` counted the 670 `{פ}/{ס}` markers as code
  letters, so its index drifted from the renderer's (which uses `stripHtml`). Codes after a parsha break didn't
  line up — e.g. a skip-1000 "תורה" hit decoded to `העוש` in the real letter stream. Only codes before the first
  marker (the famous Genesis תורה@50) happened to work.
- **`gematria()` over-counted** any verse with a `{פ}/{ס}` marker or a footnote.

## Corpus markup audit (`ingest/scan-html.mjs`)

Scanned all **4.1M** content rows. The annotation markup whose CONTENT is **not** source text:
- **footnotes — `footnote-marker` (384k) + `footnote` (385k)**: translator/masoretic notes, e.g.
  `<i class="footnote">(בספרי ספרד ואשכנז …)</i>` — by far the most common, and they carry Hebrew (so they
  polluted letter/word/gematria counts). Bodies can nest tags.
- **`mam-spi-*` parsha markers** ({פ}/{ס}/inv-nun) — already known.
Everything else wraps real text → unwrap: `b i small sup big strong em span a` and classes `poetry refLink
indentAll font1 grey rtl …`, **including `mam-kq*` ketiv/qere** (real variant text, kept).

## What shipped

- **`shared/strip.ts`**: `stripHtml()` now drops `footnote-marker`, `footnote`, and `mam-spi` **elements with
  their content** (footnote matched lazily to handle nested `<big>` etc.), then unwraps the rest. One change
  fixes every consumer (`strip`/`words`/`letters` SQL fns + previews).
- **`worker.ts`**: plugin SQL fn bodies now get `strip` (=stripHtml) and `H` in scope (trailing params, like
  `evalJS`) via `registerPluginFn`.
- **`gematria()` / `gematria_atbash()` / `torah_code_find()`**: extract via `strip(text)` + a **positive**
  `match(/[א-ת]/g)` instead of raw `replace(/[^א-ת]/g)` — so they share the one canonical cleanup. The
  torah-code matrix renderer + word parsing switched to positive-match too (it already used `stripHtml`).

## Verified (live, full Chumash + Rashi)

- **Torah letters = 304,971** (was 305,599) → **+166** over the traditional **304,805**; that +166 is the qere
  readings (ketiv-only ≈ 304,801 ≈ exact). The prior excess was footnotes/markers.
- **Corpus-spanning ELS now verifies**: תורה at skip 1000 (was `העוש`) and skip 50000 (span 150k letters across
  all 5 books) both independently decode to תורה.
- **Famous codes intact**: Genesis/Exodus תורה@50, Numbers/Deuteronomy הרות@50, Leviticus יהוה@8.
- `gematria()` Gen 1:1 = 2701, Gen 4:13 (footnoted) excludes the note; Torah words 80,016 (was 80,133).

`npx tsc` clean.
