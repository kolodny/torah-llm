# 003 — Sefaria data source moved to GCS (2026)

**Date:** 2026-06-09

## Finding

The reference project (March 2025) downloaded text data via a **git sparse-checkout** of
`github.com/Sefaria/Sefaria-Export` (`json/`, `schemas/`, `links/`, `table_of_contents.json`).

A fresh clone of that repo's `master` (commit `7c10a6a`, 2026-06-02) **no longer contains
those directories.** Per the repo's new `CLAUDE.md`:

- Text data now lives in a **public GCS bucket**: `gs://sefaria-export/` (~26 GB, ~85K files,
  regenerated monthly).
- The git repo holds only **`books.json`** — an index of metadata + download URLs for every text.

## Access pattern (no auth — public bucket)

```
https://storage.googleapis.com/sefaria-export/<path>
```

Paths:
```
table_of_contents.json
schemas/<Title>.json
json/<categories>/<Title>/<English|Hebrew>/merged.json
links/links0.csv … links12.csv
```

Verified live (HTTP 206 on range requests) for:
`table_of_contents.json`, `schemas/Genesis.json`,
`json/Tanakh/Torah/Genesis/{English,Hebrew}/merged.json`, `links/links0.csv`.

## Decision

"Fresh copies" now means **fetching the subset directly from GCS over HTTPS** — this *is* the
canonical, monthly-updated source. No multi-GB git clone. `scripts/fetch-subset.ts` downloads
exactly the files the iteration-1 subset needs into a local `sefaria/` directory mirroring the
bucket layout, so the build scripts stay simple (they just glob local files).
